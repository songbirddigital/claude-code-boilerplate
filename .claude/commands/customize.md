# Customize Boilerplate

Invoke the `boilerplate-customize` skill to adjust how the boilerplate works for this project.

## Quick Options

- `/customize greeting` — Change greeting format or disable
- `/customize defaults` — Change session settings defaults
- `/customize guardians` — Enable/disable/modify guardians
- `/customize hooks` — Add project-specific hooks
- `/customize verbosity` — Adjust how much I explain
- `/customize` — Show full customization menu

## Examples

"Make the greeting shorter"
"Disable the accessibility guardian"
"Stop showing the settings menu on startup"
"I want expert mode by default"
"Add a hook that reminds me to update docs"

## Philosophy

The boilerplate is a starting point. Customize freely.
Changes are tracked in `.boilerplate.json` so sync knows what to preserve.
